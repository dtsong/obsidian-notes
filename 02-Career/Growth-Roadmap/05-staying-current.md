# Pillar 5: Staying Current

## Overview

The AI/ML landscape moves faster than any previous technology wave. Staying current isn't optional—it's a core job requirement. This pillar provides a sustainable system for continuous learning without drowning in noise.

---

## The Challenge

**Information overload is real:**
- New model releases weekly
- Papers published daily
- Tool updates constantly
- Hype cycles obscure signal

**The goal:** Build a learning system that's:
- Sustainable (< 5 hours/week)
- High signal-to-noise
- Actionable (leads to skills, not just awareness)
- Cumulative (builds on itself)

### 5.5 Research Papers

**For Most Practitioners:**
You don't need to read papers deeply. You need to know:
- What's the key insight?
- Does it matter for my work?
- Who's talking about it?

**Sources:**

| Source | Focus | Approach |
|--------|-------|----------|
| **arXiv (cs.CL, cs.SE)** | Raw papers | Skim titles/abstracts |
| **Papers With Code** | Implementations | See what has code |
| **The Gradient** | Paper summaries | Curated explanations |
| **Two Minute Papers** | Video summaries | Quick orientation |
| **Interconnects** (Nathan Lambert) | RLHF, alignment | Deep dives on training |

**When to Read Deeply:**
- It directly impacts your current work
- Multiple credible people are discussing it
- It introduces a new paradigm (rare)

**How to Read:**
1. Abstract — What's the claim?
2. Figures — What do results show?
3. Conclusion — What do authors think matters?
4. (Only if still interested) Methods

## Learning Strategies

### 5.7 The 70/20/10 Rule

```
70% — Learning by doing (projects, work)
20% — Learning from others (reading, podcasts, conversations)
10% — Formal learning (courses, books)
```

**Implication:**
The best way to stay current is to build things. Reading supports building, but doesn't replace it.

---

### 5.8 The "Minimum Viable Follow" System

**Daily (15 min):**
- Check one newsletter (rotate)
- Skim Twitter list
- Note anything requiring deeper attention

**Weekly (1-2 hours):**
- Read 1-2 blog posts deeply
- Listen to 1 podcast episode
- Work on side project (even 30 min)

**Monthly (half day):**
- Catch up on saved articles
- Evaluate: What did I learn? What's still unclear?
- Adjust sources (add/remove)

**Quarterly:**
- Review major developments
- Update skills roadmap
- Identify next learning focus

---

### 5.9 Active Learning Techniques

**For Technical Concepts:**

| Technique | Description | When to Use |
|-----------|-------------|-------------|
| **Explain it simply** | Write a summary in plain language | New concepts |
| **Build a minimal example** | Implement the core idea | New tools/patterns |
| **Compare alternatives** | Side-by-side evaluation | Decision-making |
| **Teach someone** | Write a blog post or explain to colleague | Cementing understanding |

**The "One Thing" Rule:**
When encountering something new, identify ONE takeaway to remember and apply. Don't try to absorb everything.

---

### 5.10 Building in Public

**Why it works:**
- Forces you to understand deeply enough to explain
- Creates accountability
- Builds reputation and network
- Produces artifacts (portfolio)

**Low-friction options:**

| Format | Effort | Reach |
|--------|--------|-------|
| Twitter threads | Low | Medium |
| LinkedIn posts | Low | Medium (professional) |
| GitHub repos | Medium | High (lasting) |
| Blog posts | Medium | High (SEO) |
| Conference talks | High | High (credibility) |

**Starter Ideas:**
- "What I learned implementing X"
- "Comparison: Tool A vs Tool B for Y use case"
- "Mistakes I made with Z and how I fixed them"
- Code snippets with explanation

## Avoiding Burnout

### 5.13 Sustainable Practices

**Accept You'll Miss Things:**
- You cannot follow everything
- FOMO is the enemy of focus
- Missing a trend by a month doesn't matter

**Quality Over Quantity:**
- One deep dive > ten skimmed articles
- One completed project > five abandoned ones
- One strong relationship > fifty weak connections

**Scheduled Breaks:**
- It's okay to go dark for a week
- The important stuff will still be there
- Your subconscious processes in the background

**Recognize Diminishing Returns:**
- After 5 hours/week, returns drop sharply
- Beyond staying informed, focus on building
- Information consumption is not productivity

## Key Takeaways

1. **Build a system, not a habit** — Systems survive schedule disruptions
2. **High signal > high volume** — Curate aggressively
3. **Projects > consumption** — Building teaches more than reading
4. **Accept incompleteness** — You'll never know everything, and that's fine
5. **Teach to learn** — Writing/explaining cements understanding
6. **Community accelerates** — Learn with others when possible
7. **Sustainability matters** — 5 hours/week for years > 20 hours/week for months
